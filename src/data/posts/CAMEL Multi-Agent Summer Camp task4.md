# CAMEL Multi-Agent Summer Camp Task4
在[CAMEL Multi-Agent Summer Camp Task3](./camel-multi-agent-summer-camp-task3)中我们已经了解了CAMEL的 workforce 用法，在本文我将用 workforce 构建一个辩论赛多智能体系统，然后利用几个常见的辩题进行测试。
代码如下：
```python
from camel.agents import ChatAgent
from camel.models import ModelFactory
from camel.types import ModelPlatformType, ModelType
from camel.messages import BaseMessage
from camel.societies.workforce import Workforce
from camel.toolkits import SearchToolkit
from camel.tasks import Task
from camel.toolkits import FunctionTool

import os
from dotenv import load_dotenv

load_dotenv()

MODELSCOPE_API_KEY = os.getenv("MODELSCOPE_API_KEY")

model = ModelFactory.create(
    model_platform=ModelPlatformType.OPENAI_COMPATIBLE_MODEL,
    model_type="Qwen/Qwen3-235B-A22B-Instruct-2507",
    url='https://api-inference.modelscope.cn/v1/',
    api_key=MODELSCOPE_API_KEY,
)

workforce = Workforce(
    description="辩论赛团队",
    coordinator_agent=ChatAgent(
        system_message="""
你是辩论赛的协调员，负责组织和协调辩论赛的各个环节。
辩论赛流程：
1. 正方选手第一轮发言
2. 反方选手第一轮发言
3. 正方选手第二轮发言
4. 反方选手第二轮发言
5. 正方选手第三轮发言
6. 反方选手第三轮发言
7. 正方选手第四轮发言
8. 反方选手第四轮发言
9. 记录员记录所有选手发言要点
10. 裁判总结所有选手发言并给出最终评判
协调职责：
1.  严格确认发言轮数，由正方开始发言，始终保持正方先发言、反方后发言，轮流发言的顺序
2.  禁止连续安排同一方发言
""",
        model=model,
    ),
    task_agent=ChatAgent(
        system_message="""
你是辩论赛的任务处理者，负责分配和管理辩论赛的各项任务。
分配任务：
1.  第一轮发言：陈述己方观点，列举事实进行支持
2.  第二轮发言：针对对方观点进行反驳，提供事实和数据支持
3.  第三轮发言：针对对方观点进行反驳，证明己方观点
4.  第四轮发言：总结己方观点，强调立场
5.  记录员任务：总结四轮发言内容
6.  裁判任务：给出最终评判
""",
        model=model
    )
)

search_tool = FunctionTool(
    func=SearchToolkit().search_baidu
)

pro_agent = ChatAgent(
    system_message="""
你是一个辩论赛团队的成员，你作为正方成员，你需要提供有力的论据和事实支持你的立场。
发言规则：
1. 你必须先陈述你的立场和观点，你最先发言
2. 你需要提供事实和数据来支持你的论点
3. 你必须尊重对方的观点，并进行合理的反驳
4. 你必须在发言结束时总结你的观点
5. 最多进行4轮发言，每轮发言时首先标注发言轮数，4轮发言后不可再进行发言
6. 你的发言轮数为1-4，轮次必须依次进行，你的发言轮数为你的上一轮发言轮数加1
发言格式：
\`\`\`
第1轮发言：
<发言内容>
\`\`\`
""",
    model=model,
    tools=[search_tool],
    output_language="中文"
)

con_agent = ChatAgent(
    system_message="""
你是一个辩论赛团队的成员，你作为反方成员，你需要提供有力的论据和事实支持你的立场。
发言规则：
1. 你必须先陈述你的立场和观点
2. 你需要提供事实和数据来支持你的论点
3. 你必须尊重对方的观点，并进行合理的反驳
4. 你必须在发言结束时总结你的观点
5. 最多进行4轮发言，每轮发言时首先标注发言轮数，4轮发言后不可再进行发言
6. 你的发言轮数为1-4，轮次必须依次进行，你的发言轮数和对方的发言轮数相同
发言格式：
\`\`\`
第1轮发言：
<发言内容>
\`\`\`
""",
    model=model,
    tools=[search_tool],
    output_language="中文"
)

record_agent = ChatAgent(
    system_message="""
你是一个辩论赛团队的记录员，你需要记录整个辩论赛的过程，包括每个发言者的观点、论据和反驳。
记录规则：
1. 你必须准确记录每个发言者的立场和观点，不带任何个人情感
2. 你需要记录每个发言者提供的事实和数据
3. 你必须记录每个发言者对对方观点的反驳
4. 你必须在第4轮发言结束时总结每个发言者的观点
5. 你必须分点记录内容，采用Markdown格式
6. 你必须在最后一轮发言(第4轮)结束后发言
""",
    model=model,
    output_language="中文"
)

referee_agent = ChatAgent(
    system_message="""
你是一个辩论赛的裁判，你需要公正地评判每个发言者的观点和论据。
裁判规则：
1. 你必须准确理解每个发言者的立场和观点
2. 你需要评估每个发言者提供的事实和数据的有效性
3. 你必须公正地对待每个发言者的观点，并给出合理的评判
4. 你必须在记录员发言结束后发言
5. 你必须根据双方的表现给出最终的评判
6. 你发言后任务完成
""",
    model=model,
    output_language="中文"
)

workforce.add_single_agent_worker(
    description="正方辩手，支持正方观点",
    worker=pro_agent
).add_single_agent_worker(
    description="反方辩手，支持反方观点",
    worker=con_agent
).add_single_agent_worker(
    description="记录员，记录辩论过程",
    worker=record_agent
).add_single_agent_worker(
    description="裁判，评判辩论结果",
    worker=referee_agent
)

task = Task(
    content="进行一场辩论赛，主题是‘顺境还是逆境更有利于个人成长’", # 可以设置为不同的辩题
    additional_info={
        "规则": """1. 正方先发言，反方后发言，轮流发言
2. 每个发言者最多进行4轮发言
3. 第一轮发言：开篇立论，明确己方立场，提出己方核心论点
4. 第二、三轮发言：攻辩/质询。攻方针对对方提出观点中的漏洞和不足提出质疑，同时强调自身观点；守方回答问题，坚定自身立场防止被带偏。每一轮发言后攻防互换。
5. 第四轮发言：总结陈词，梳理辩论内容，再次指出对方核心漏洞，强调己方观点
6. 记录员记录每个发言者的观点和论据
7. 裁判在记录员记录结束后给出最终评判
"""
    },
    id="辩论赛-1"
)

print("Starting task...")
task_res = workforce.process_task(task)
print("Task completed.")
print(task_res.result)
```
在这个辩论赛多智能体系统中，我们精心设计了六个核心角色，每个角色都有明确的职责定位和协作方式，共同推动辩论流程有序进行： 
1. **正方辩手**：坚定坚定支持辩题中的正面立场，拥有网络搜索能力以获取事实数据支撑观点。按照规则需完成四轮发言，首轮明确阐述核心论点及依据，后续几轮针对反方观点进行针对性反驳，同时深化己方论证，最终轮次进行总结升华，每轮发言均需清晰标注轮次。 
2. **反方辩手**： 与正方辩手形成对抗，支持辩题中的反面立场，同样具备搜索工具使用权限。严格遵循"正方先发言、反方后回应"的顺序，四轮发言分别对应开篇立论、针对性反驳、强化论证和总结陈词，确保与正方保持相同的轮次节奏。 
3. **记录员**： 全程客观记录辩论过程，不加入个人观点。详细记录双方每轮发言的核心论点、关键论据（包括引用的事实和数据）以及相互反驳的要点，采用分点列明的Markdown格式呈现。在双方完成四轮辩论后，对整体内容进行系统性梳理总结。
4. **裁判**： 作为辩论结果的最终评判者，需全面理解双方的立场和论证逻辑。重点评估双方论据的真实性、有效性以及反驳的针对性，在记录员完成总结后，依据双方的整体表现给出公正的评判结果，裁判发言后整个辩论任务即宣告结束。
5. **协调员**： 负责把控辩论的整体流程和节奏，确保严格遵循预设的发言顺序。从正方首轮发言开始，依次协调反方回应，全程维持"正方-反方"交替发言的规则，杜绝同一方连续发言的情况，保障辩论环节有序推进。 
6. **任务处理者**： 作为任务分配中枢，根据辩论进程向各角色下达具体任务。例如指导首轮发言需完成立论，二、三轮聚焦攻防辩论，四轮进行总结；同时明确记录员的总结任务和裁判的评判任务，确保每个角色清楚知晓不同阶段的工作重点。 
这些角色通过Workforce框架有机结合，形成一个分工明确、协作紧密的辩论系统，能够按照规范的辩论流程完成从立论到评判的全环节任务。

在**反驳有效性**方面，正方精准抓住反方核心漏洞——概念混淆。通过指出‘挫折教育’‘项目失败’等所谓‘逆境’实则发生在家庭、学校等顺境保障之下，成功揭示反方论据反而支持己方立场，实现‘以子之矛攻子之盾’。反方虽有效回应‘幸存者偏差’质疑，强调林肯、乔布斯案例的普遍启示意义，但对其论据所依赖的顺境背景缺乏自洽解释，削弱了反驳力度。

综上，正方在逻辑严密性与反驳有效性上略胜一筹，其论证更全面地揭示了成长的普遍条件与系统性基础，强调教育、医疗、家庭支持等结构性因素的重要性，更具现实关怀与政策指导意义。反方虽在案例生动性与理论深度上表现不俗，但未能摆脱对顺境前提的隐性依赖。

最终评判：正方‘顺境更有利于个人成长’的立场在整体论证中更具说服力。
